# 书灵 BookSage 智能图书推荐系统项目报告

## 一、项目背景

图书数量持续增长后，读者很难仅靠分类或榜单快速找到真正感兴趣的书。本项目围绕“智能图书推荐”场景，基于 Book-Crossing 数据集和机器学习推荐模型，实现一个可搜索、可推荐、可展示的中文图书推荐平台。

## 二、项目目标

- 提供中文化的图书推荐使用界面。
- 支持热门图书展示、图书搜索和推荐结果展示。
- 保留协同过滤、内容相似和混合推荐三类推荐能力。
- 使用 FastAPI 和 React 构建前后端分离系统。
- 支持 Docker Compose 一键启动，方便验收和演示。

## 三、系统设计

系统分为前端展示层、后端接口层和推荐模型层。

前端使用 React 和 Tailwind CSS 构建，负责首页展示、搜索交互、推荐方式选择和结果展示。后端使用 FastAPI 提供 JSON API，负责加载模型、查询图书数据和调用推荐引擎。推荐模型层复用已训练的协同过滤、内容相似和混合推荐模型。

## 四、核心功能

| 功能 | 说明 |
| --- | --- |
| 热门图书 | 展示样本数据中较热门的图书 |
| 图书搜索 | 根据标题关键词查找图书 |
| 协同过滤 | 根据相似读者的行为推荐图书 |
| 内容相似 | 根据图书文本和元数据相似度推荐图书 |
| 混合推荐 | 融合协同过滤和内容相似结果 |
| 推荐解释 | 在结果页展示推荐方式说明和推荐强度 |

## 五、推荐方式说明

协同过滤侧重读者行为，适合发现“相似读者也喜欢”的图书。内容相似侧重图书特征，适合发现题材、作者或出版信息相近的作品。混合推荐综合两类方法，是系统默认推荐策略。

## 六、运行与部署

项目提供 Docker Compose 配置，后端服务默认运行在 `8000` 端口，前端服务默认运行在 `3000` 端口。部署后可通过前端页面完成完整推荐流程，也可以直接访问 `/api/health` 检查后端服务状态。

## 七、验收结果

- 前端主要用户可见文案完成中文化。
- 推荐方式按钮、结果页说明、空状态和加载状态均为中文。
- README 已改写为中文提交版说明。
- FastAPI 标题、描述和健康检查响应增加中文项目标识。
- 保持原有推荐接口参数兼容：`hybrid`、`collaborative`、`content`。

## 八、后续优化

后续可以继续补充运行截图、演示 PPT、答辩讲稿，并增加用户登录、收藏记录和推荐历史等功能，使系统更接近真实图书推荐产品。
